5 Nov 2018 (RC5)

New Features

Improvements

  • Synchronisation state is now persisted against a given instance so that when updating synchronisation it will pre-select all items that are already being synced.
  • Studio honours the flag Automatically sync any new application that appears in the Platform Instance. The flag will ensure that as new integrations are added in OIC they will be automatically synced to MyST.
  • Failure and success status for synchronisation is clearly shown by the wizard for a brief period after running the synchronisation.
  • Developer Cloud Service is now a valid Repository option in the Continuous Delivery Profile
  • There is now an option for selecting or deselecting all in synchronisation wizard and an option to reset synchronisation to the selection from the last successful run
  • Continuous Delivery Profile is now used as the source of Git and Maven Repository for synchronisation

Bug Fixes

  • Synchronisation will continue processing in the case of individual artifact synchronisation errors rather than failing the whole synchonisation
